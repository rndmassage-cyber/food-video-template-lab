# ADR-0001: Bootstrap репозитория food-video-template-lab

## Status

Accepted

## Date

2026-03-24

## Context

Проект food-video-template-lab создаётся как R&D-лаборатория для проектирования стабильных шаблонов короткой food-video анимации. Нужна управляемая структура репозитория, в которой можно последовательно развивать template-driven workflow — от проектирования до production handoff.

Основная работа ведётся с помощью Claude Code, поэтому репозиторий должен быть самодокументирован: агент должен иметь возможность прочитать контекст проекта из файлов, а не полагаться только на chat-историю.

## Decision

1. Создать единый репозиторий проекта со следующей структурой:
   - `docs/` — документация, roadmap, ADR, runbooks
   - `templates/` — спецификации шаблонов, prompt packs, test suites
   - `fixtures/` — входные и референсные материалы для тестов
   - `runs/` — результаты запусков и производные R&D-артефакты, которые обычно не хранятся в git (кроме .gitkeep при необходимости)
   - `scripts/` — служебные скрипты
2. Зафиксировать проектные правила в `CLAUDE.md` как основной контекст для Claude Code.
3. Использовать `README.md` и `docs/project-map.md` как дополнительные точки входа.
4. Использовать ADR-формат для архитектурных решений.
5. Не усложнять структуру раньше времени — начать с минимального набора папок и документов.

## Consequences

- Claude Code может ориентироваться в проекте, читая файлы из репозитория.
- Структура готова к поэтапному наращиванию без рефакторинга на ранних фазах.
- Ограничения V1 зафиксированы в нескольких документах для надёжности контекста.
- Все решения прозрачны и отслеживаемы через ADR.
