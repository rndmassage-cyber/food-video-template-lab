# ADR-0002: Границы и ограничения V1

## Status

Proposed

## Date

2026-03-24

## Context

Проект food-video-template-lab строится как R&D-лаборатория шаблонов food-video анимации.
Перед началом проектирования сущностей и архитектуры необходимо формально зафиксировать
границы первой версии (V1) в одном авторитетном месте.

До этого ADR V1-ограничения были описаны повествовательно и продублированы в нескольких
файлах (CLAUDE.md, README.md, project-map.md, roadmap.md). Это создаёт риск рассинхронизации
и неоднозначных трактовок при дальнейшей работе.

Данный ADR становится canonical source of truth для V1 scope.
Остальные документы могут содержать краткое summary ограничений, но при конфликте
приоритет у этого ADR.

## Decision

### Входные данные

1. Используются только **внутренние изображения** из собственного image-pipeline.
2. Внешние пользовательские загрузки в V1 не рассматриваются.
3. На изображении всегда **одно блюдо**.
4. Фон всегда **однотонный**.
5. Блюдо находится в **safe area** с контролируемым margin.

### Ракурсы

6. Допустимы только 3 семейства ракурсов:
   - `top_down`
   - `three_quarter`
   - `hero_side`
7. Не все комбинации template family × angle family обязаны быть допустимыми.
   Конкретная матрица совместимости определяется в `docs/domain-model.md`.

### Video generation

8. Генерация видео выполняется через **Kling 3.0 via Kie AI API**.
9. Режим генерации в V1:
   - `single-shot`
   - `start frame only`
   - `mode=pro`
   - `sound=false`
   - `multi_shots=false`

### Параметры, выбираемые пользователем

10. Пользователь выбирает:
    - `template` — шаблон анимации;
    - `aspect_ratio` — одно из: `1:1`, `9:16`, `16:9`;
    - `duration` — длительность ролика.
11. **Duration policy.**
    - Kling 3.0 поддерживает configurable duration в диапазоне **3–15 секунд**.
    - Проектный V1 policy range: **5–15 секунд**.
    - Конкретные `supported_durations` для каждого template определяются
      отдельно на основании тестирования. Template не обязан поддерживать
      весь policy range — только те значения, которые подтверждены test runs.

### Precompose

12. Aspect ratio обеспечивается через **precompose start frame**, а не через
    внутреннюю адаптацию Kling.

### Template ↔ Prompt pack

13. В V1 один template связан ровно с одним prompt pack (связка **1:1**).
    A/B тестирование через несколько prompt packs для одного template — за рамками V1.

### Стартовые template families

14. На первом этапе фокус на 4 семействах:
    - `orbit_micro`
    - `dolly_in_clean`
    - `float_parallax`
    - `light_accent`

## Явные исключения из V1

Следующие возможности сознательно не входят в V1:

| Исключение | Обоснование |
|---|---|
| Multi-shot | Добавляет сложность в оркестрацию и тестирование. Начинаем с single-shot. |
| Audio | Ортогонален основной задаче. Можно добавить позже без архитектурных изменений. |
| End frame | Увеличивает пространство вариативности. Start frame only проще контролировать. |
| Hand / waiter scenes | Требует дополнительных входных данных и CV-логики. |
| Background replacement | Противоречит ограничению на однотонный фон. |
| Angle conversion | Требует тяжёлого CV. В V1 ракурс задаётся исходным изображением. |
| Тяжёлый CV / vision analysis | Использовать только если без него нельзя обойтись. |
| Multi-agent orchestration | Преждевременная архитектурная сложность. |
| Plugin-heavy / framework-heavy подход | Риск хрупкости на раннем этапе. |

## Consequences

**Что получаем:**
- Canonical source of truth для V1 scope вместо дублирования в 4 файлах.
- Чёткие границы для проектирования сущностей и schema (Фазы 1–2).
- Контролируемая сложность: один shot, один start frame, один prompt pack на template.
- Основа для проектирования domain model и template schema.

**Чем платим:**
- Ограниченная гибкость в V1: нет A/B тестирования prompt packs, нет multi-shot.
- Пользователь не загружает свои изображения.
- Нет audio-layer — ролики только визуальные.
- Матрица совместимости template × angle финализируется
  по результатам тестирования (Phase 6).

## Условия пересмотра

Этот ADR может быть пересмотрен, если:
- V1-контур стабилизирован и передан в production (Фаза 8);
- появилась подтверждённая потребность в расширении (multi-shot, audio, user uploads);
- расширение не ломает существующие стабильные шаблоны.

Любое изменение V1 scope оформляется как новый ADR или amendment к данному.
