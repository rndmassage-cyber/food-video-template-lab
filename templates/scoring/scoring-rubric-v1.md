# Scoring Rubric V1 — food-video-template-lab

## Статус

Proposed (Phase 2, batch 3)

## Дата

2026-03-25

## Назначение

Определяет методику оценки результатов генерации видео для V1.

Scoring rubric отвечает на вопросы:
- По какой шкале оценивать?
- Что означает каждый балл для каждого критерия?
- Как определить общий verdict по run?
- Как агрегировать verdicts на уровне test suite?

Canonical sources: test-suite-format.md (scoring_criteria), domain-model.md (сущность score),
ADR-0003 (И-4: score → run).

---

## Решение: единая rubric для V1

**Вопрос из domain-model.md (open Q#3):** единая scoring rubric или per-template?

**Ответ для V1:** единая rubric.

**Обоснование:**
- В V1 только 4 template families с похожей базовой структурой
  (одно блюдо, однотонный фон, одно движение камеры).
- Все 5 стандартных критериев применимы ко всем templates.
- Template-specific акценты покрываются полем `scoring_hints` в template spec
  и полем `notes` в criteria_results run record.
- Единая rubric = единый стандарт сравнения между templates.

**Когда пересмотреть:** если появятся templates с принципиально иной structure
(например, multi-shot, scene transitions), потребуются дополнительные критерии
или отдельные rubrics.

## Метод оценки в V1

**Evaluator:** human (ручной просмотр видео).

**Процесс:**
1. Оценщик просматривает видео целиком.
2. Оценивает каждый criterion по шкале 1–5.
3. Записывает комментарии к каждому criterion (опционально, но рекомендуется).
4. Определяет overall_verdict по правилам ниже.

**Контекст при оценке:** оценщик должен иметь перед глазами:
- motion_intent из template spec (что ожидалось);
- scoring_hints из template spec (на что обращать внимание);
- исходное изображение (start frame).

## Шкала: 1–5

| Балл | Уровень | Общее описание |
|------|---------|----------------|
| **5** | Отлично | Нет замечаний. Результат полностью соответствует intent. Готов к production. |
| **4** | Хорошо | Минорные замечания, не влияющие на восприятие. Приемлем для production. |
| **3** | Приемлемо | Заметные проблемы, но intent в целом выполнен. Требует внимания. |
| **2** | Плохо | Серьёзные дефекты. Intent частично провален. Не production-ready. |
| **1** | Провал | Фундаментальная проблема. Intent не выполнен или результат неприемлем. |

## Критерии оценки

### dish_integrity (weight: critical)

Сохранность формы, пропорций и деталей блюда на протяжении всего видео.

| Балл | Описание |
|------|----------|
| 5 | Блюдо идентично начальному кадру от начала до конца. Все детали сохранены. |
| 4 | Минимальные отклонения: лёгкое смещение деталей, не заметное без покадрового сравнения. |
| 3 | Заметные изменения формы или деталей, но блюдо узнаваемо. Например: сдвиг элементов, лёгкое «плавание» текстуры. |
| 2 | Серьёзная деформация: блюдо меняет форму, элементы сливаются или дублируются. |
| 1 | Блюдо неузнаваемо, «тает», морфирует или исчезает. |

### motion_stability (weight: critical)

Стабильность и плавность движения камеры.

| Балл | Описание |
|------|----------|
| 5 | Абсолютно плавное движение на всём протяжении. Нет дрожи, рывков, пауз. |
| 4 | Преимущественно плавное. Минимальные отклонения (1–2 микро-рывка), незаметные при нормальном просмотре. |
| 3 | Заметные нестабильности: периодическая дрожь, неравномерная скорость, но общее движение сохраняется. |
| 2 | Серьёзные проблемы: множественные рывки, замирания, внезапные ускорения. Движение некомфортно для зрителя. |
| 1 | Движение хаотично, отсутствует или прерывается. Видео непригодно для использования. |

### motion_intent_match (weight: high)

Соответствие фактического движения описанному в template.motion_intent.

| Балл | Описание |
|------|----------|
| 5 | Движение точно соответствует intent: тип, направление, амплитуда. |
| 4 | Движение соответствует intent с незначительными отклонениями (например, чуть меньше/больше градусов). |
| 3 | Движение в целом правильного типа, но с заметными отклонениями: другая амплитуда, скорость или акцент. |
| 2 | Движение частично соответствует intent, но доминирует другой тип движения (например, вместо orbit — zoom). |
| 1 | Движение не соответствует intent: статичный кадр, или совершенно другое движение. |

### background_stability (weight: high)

Чистота и стабильность фона на протяжении видео.

| Балл | Описание |
|------|----------|
| 5 | Фон абсолютно стабилен: цвет, текстура, освещённость неизменны. |
| 4 | Минимальные изменения фона: лёгкий shift освещения или тона, незаметный без сравнения. |
| 3 | Заметные изменения: мерцание, градиент, лёгкие текстурные артефакты на фоне. |
| 2 | Серьёзные проблемы: появление теней, пятен, текстур. Фон перестаёт быть однотонным. |
| 1 | Фон кардинально меняется: появляются объекты, сцена, цвет фона изменился. |

### no_artifacts (weight: normal)

Отсутствие визуальных глитчей, дублирования, размытий и прочих артефактов генерации.

| Балл | Описание |
|------|----------|
| 5 | Визуальных артефактов нет. Видео выглядит как снятое камерой. |
| 4 | 1–2 минимальных артефакта (лёгкое размытие на краю, микро-глитч), не влияющие на восприятие. |
| 3 | Несколько заметных артефактов, но они не доминируют. Видео остаётся смотрибельным. |
| 2 | Множественные артефакты: дублирование элементов, «призраки», мерцание. Видео непригодно для production. |
| 1 | Артефакты доминируют. Видео визуально сломано. |

## Определение overall_verdict

Overall verdict для одного run определяется по следующим правилам,
применяемым **последовательно** (первое сработавшее правило определяет verdict):

### Правило 1: Fail (немедленный)

Если **любой** критерий с weight=critical имеет score ≤ 2 → **fail**.

### Правило 2: Fail (серьёзный дефект)

Если **любой** критерий с weight=high имеет score = 1 → **fail**.

### Правило 3: Pass

Если **все** условия выполнены:
- все critical criteria ≥ 3
- все high criteria ≥ 3
- weighted average ≥ 3.5

→ **pass**.

### Правило 4: Marginal

Если **все** условия выполнены:
- все critical criteria ≥ 3
- все criteria ≥ 2
- weighted average ≥ 2.5

→ **marginal**.

### Правило 5: Fail (default)

Во всех остальных случаях → **fail**.

### Расчёт weighted average

```
weighted_average = Σ(score_i × weight_i) / Σ(weight_i)

weight values:
  critical = 3
  high     = 2
  normal   = 1
```

Для стандартного набора V1 (2 critical + 2 high + 1 normal):

```
weighted_average = (dish_integrity×3 + motion_stability×3 + motion_intent_match×2 + background_stability×2 + no_artifacts×1) / 11
```

### Пример расчёта

Из примера run record (orbit_micro_default):
- dish_integrity: 5 × 3 = 15
- motion_stability: 4 × 3 = 12
- motion_intent_match: 4 × 2 = 8
- background_stability: 5 × 2 = 10
- no_artifacts: 4 × 1 = 4

Сумма: 49 / 11 = **4.45**

Проверка правил:
- Правило 1: нет critical ≤ 2 → не fail
- Правило 2: нет high = 1 → не fail
- Правило 3: все critical ≥ 3, все high ≥ 3, 4.45 ≥ 3.5 → **pass**

## Агрегация на уровне test suite

Test suite определяет `min_pass_rate` (proposed default: 0.7).

**Формула:**

```
pass_rate = count(runs с verdict=pass) / count(scored runs)
```

**Рекомендация по решению о template:**

| pass_rate | Рекомендация |
|-----------|-------------|
| ≥ min_pass_rate | Template пригоден к продвижению (candidate / approved) |
| ≥ 0.5 и < min_pass_rate | Пограничный результат. Анализ failed/marginal runs, возможна доработка |
| < 0.5 | Template нестабилен. Требуется revision или пересмотр подхода |

**Роль marginal runs:** runs с verdict=marginal НЕ считаются pass.
Они сигнализируют о пограничном качестве и требуют внимания.

## Template-specific дополнения

Scoring rubric V1 — единая для всех templates.
Template-specific акценты передаются через:

1. **template.scoring_hints** — подсказки, на что обращать внимание
   (например: «у orbit_micro важна стабильность краёв блюда при вращении»).
2. **test_suite.scoring_criteria.description** — уточнённые описания критериев
   для конкретного template.
3. **criteria_results.notes** в run record — комментарии оценщика
   с учётом template-specific контекста.

Rubric определяет **шкалу и правила**, template определяет **акценты**.

## Что rubric НЕ определяет

| Исключённое | Почему |
|---|---|
| Автоматический scoring pipeline | За рамками V1. V1 = ручная оценка |
| LLM-judge prompts | Если LLM-judge будет использоваться, это отдельная спецификация |
| Сравнительный scoring (A vs B) | V1 оценивает каждый run абсолютно, не относительно |
| Весовые коэффициенты per-template | V1 использует единые weights |
| Повторяемость (re-test) scoring | Определяется позже при необходимости |
